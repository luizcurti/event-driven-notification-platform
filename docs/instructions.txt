# Project: Event-Driven Notification Platform

## Project Name

`event-driven-notification-platform`

---

# Objective

Build a notification platform based on an **Event-Driven Architecture (EDA)**.

The system must receive events from multiple external systems and automatically determine which communication channels should be triggered.

### Example Events

* OrderApproved
* UserRegistered
* PaymentFailed
* PasswordChanged
* DocumentProcessed

Each event may trigger one or more notification channels.

```text
OrderApproved
       |
       |
  EventBridge
       |
+------+------+------+------+
|      |      |      |
Email  SMS   Push  Webhook
```

The primary goal is to demonstrate:

* Event-Driven Architecture
* Fan-out Pattern
* Serverless Architecture
* Loose Coupling
* Retry Mechanism
* Observability
* Infrastructure as Code (IaC)
* Clean Architecture without overengineering

# Core Principle

The system must **never** know its consumers directly.

### Incorrect

```text
API Lambda
   |
   +--> Email Service
   |
   +--> SMS Service
   |
   +--> Push Service
```

This creates tight coupling.

### Correct

```text
API Lambda
    |
    |
EventBridge
    |
    +--> Email
    |
    +--> SMS
    |
    +--> Push
```

The producer only publishes events.

Consumers independently decide whether they should process them.

# Domain Model

Create the `Notification` domain.

```typescript
Notification {
  id: string
  eventType: string
  recipient: string
  channels: string[]
  payload: any
  status: NotificationStatus
  createdAt: Date
  updatedAt: Date
}
```

## Notification Status

```typescript
enum NotificationStatus {
  PENDING,
  PROCESSING,
  DELIVERED,
  FAILED,
  RETRYING,
  EXPIRED
}
```
